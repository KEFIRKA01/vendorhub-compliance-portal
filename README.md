`Портал комплаенса поставщиков VendorHub` показывает B2B-контур онбординга поставщиков: документы, оценку риска, маршруты согласования и контроль зависших карточек в одном управляемом процессе.

## Что показывает проект

- supplier onboarding и прохождение поставщика через проверку;
- комплаенс-скоринг и выявление рискованных кейсов;
- матрицу согласования с участием нескольких ролей;
- эскалацию по зависшим карточкам и проблемным маршрутам;
- международные и повышенно рискованные сценарии;
- архитектуру для B2B-порталов, vendor management и внутренних compliance-систем.

## Для каких задач подходит

- управление поставщиками и подрядчиками;
- B2B-кабинеты и внутренние панели проверки;
- комплаенс, документооборот и approval flow;
- onboarding-порталы для закупок и партнёрских сетей;
- проекты, где важны прозрачные статусы и контроль блокеров.

## Ключевые сценарии

- старт проверки нового поставщика;
- расчёт риска по документам и атрибутам компании;
- прохождение карточки через несколько команд согласования;
- эскалация при зависании на этапе проверки;
- обработка нестандартного международного или высокорискованного кейса.

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

## Почему это сильный кейс

- отлично раскрывает направление `B2B + compliance + vendor management`;
- показывает сильную внутреннюю бизнес-логику, а не только форму и таблицу;
- помогает продавать заказы про поставщиков, проверки, маршруты согласования и сложные корпоративные процессы.
